\chapter{Bevezetés}

\section{A téma aktualitása és célja}

A modern vállalati környezetben az informatikai eszközök -- mint például számítógépek, mobiltelefonok, nyomtatók vagy SIM-kártyák -- kulcsfontosságú szerepet töltenek be a mindennapi munkavégzésben. A digitalizáció térnyerésével és a távmunka elterjedésével az eszközök száma, valamint azok nyilvántartásának bonyolultsága is jelentősen megnőtt. Egy közepes méretű cég esetében akár több száz különböző eszköz mozgását, státuszát és felelősét kell naprakészen kezelni, ami manuálisan, például Excel-táblák segítségével, már nehezen megvalósítható.

A szakdolgozat célja egy olyan webalkalmazás fejlesztése, amely átlátható és automatizált módon kezeli az informatikai eszközök nyilvántartását és kiadását. A rendszer segítségével a felhasználók egyszerűen igényelhetnek eszközöket, a jóváhagyás és a kiadás folyamata pedig digitálisan nyomon követhetővé válik. Ezáltal csökken az adminisztrációs teher, és minimalizálható az emberi hibákból eredő adateltérés.

A megoldás technológiai alapját a .NET 8 Web API adja, amely egy korszerű, gyors és biztonságos szerveroldali keretrendszer, ideális vállalati szintű alkalmazások fejlesztéséhez. A felhasználói felület React alapokra épül, amely a modern JavaScript-ökoszisztéma egyik legnépszerűbb könyvtára, dinamikus és reszponzív webes felületek készítéséhez.

A fejlesztés célkitűzései közé tartozik:

az informatikai eszközök (pl. számítógépek, mobiltelefonok) nyilvántartásának központi kezelése,

a kiadási és visszavételi folyamatok digitalizálása,

a felhasználói és vezetői jogosultságok kezelése,

valamint a jóváhagyási folyamat beépítése a rendelési rendszerbe.

Egy ilyen rendszer bevezetése növeli a vállalat átláthatóságát, támogatja az eszközgazdálkodás tervezhetőségét, és hosszú távon csökkenti az üzemeltetési költségeket. A fejlesztés személyes célja továbbá az, hogy elmélyítsem ismereteit a modern webfejlesztési technológiákban és az alkalmazásfejlesztés vállalati szintű gyakorlatában.

\section{A használt technológiák bemutatása}
Az alkalmazás fejlesztése során a cél egy olyan modern, megbízható és hosszú távon fenntartható technológiai alap kialakítása volt, amely illeszkedik a jelenlegi iparági trendekhez és a vállalati fejlesztési környezetek elvárásaihoz. A választott technológiák — .NET 8 Web API a backend oldalon, valamint React a frontend oldalon — egyaránt támogatják a gyors fejlesztést, a biztonságos működést és az egyszerű bővíthetőséget.

\subsection{.NET 8 Web API}

A szerveroldali logika megvalósításához a .NET 8 Web API keretrendszert alkalmaztam, amely a Microsoft által fejlesztett, nyílt forráskódú platform legfrissebb verziója. A .NET 8 előnye, hogy támogatja a minimal API-k használatát, így egyszerű, mégis hatékony módon lehet REST alapú végpontokat létrehozni. A rendszer architektúrája rétegzett felépítést követ: az adat-hozzáférési réteg (Repository pattern) kezeli az adatbázissal való kommunikációt, míg az API réteg felel a kliensoldali kérések kiszolgálásáért. Az alkalmazásban az Entity Framework Core (EF Core) ORM (Object-Relational Mapping) keretrendszer gondoskodik az adatbázis műveletek egyszerű és biztonságos kezeléséről. Az EF Core segítségével LINQ lekérdezések használatával lehet az adatokhoz hozzáférni, ezáltal a kód átláthatóbb és karbantarthatóbb (Microsoft Learn, 2023). Az adatok tárolása Microsoft SQL Server adatbázisban történik, amely jól integrálható a .NET ökoszisztémával és alkalmas nagyvállalati környezetben való használatra is. A felhasználók azonosítását és jogosultságkezelését JWT (JSON Web Token) alapú autentikáció biztosítja, amely a kliens és a szerver között biztonságos módon kezeli a bejelentkezési adatokat.

\subsection{React Frontend}

A kliensoldali felület React segítségével készült, amely a Meta (Facebook) által fejlesztett JavaScript-könyvtár, és jelenleg a legelterjedtebb frontend-technológiák közé tartozik (Stack Overflow Developer Survey, 2024). A React komponensalapú megközelítése lehetővé teszi, hogy az alkalmazás logikailag jól szeparált, újrahasznosítható elemekből épüljön fel, így a fejlesztés és a karbantartás is hatékonyabbá válik. Az alkalmazásban React Router gondoskodik a kliensoldali navigációról, míg az Axios könyvtár biztosítja az API-hívásokat a backend felé. A fejlesztés során külön figyelmet fordítottam a felhasználói élményre és az ergonómiára, ezért a frontend kialakításához Tailwind keretrendszert használtam, amely modern, reszponzív és mobilbarát megjelenést biztosít.

\subsection{Fejlesztési környezet és eszközök}
A fejlesztés Visual Studio 2022 és Visual Studio Code környezetben zajlott, amelyek hatékony támogatást nyújtanak a .NET és JavaScript alapú projektekhez. A verziókezeléshez Git rendszert használtam, a forráskódot pedig GitHub tárhelyen tároltam.

\section{A rendszer célja és funkcionális követelményei}

Az informatikai eszközök nyilvántartására és kiadásának kezelésére tervezett rendszer célja, hogy \textbf{átlátható, digitálisan kezelhető folyamatot} biztosítson a vállalati IT-eszközök kezeléséhez. 
A fejlesztés során fontos szempont volt, hogy az alkalmazás \textbf{egyszerűen használható, biztonságos és könnyen bővíthető} legyen, valamint hogy a napi adminisztrációs feladatokból minél többet automatizáljon.

A vállalatoknál jellemző probléma, hogy az eszköznyilvántartás különböző táblázatokban, dokumentumokban történik. 
Ez könnyen vezethet \textbf{adatvesztéshez, átláthatatlan folyamatokhoz és hibás eszköz-hozzárendelésekhez}. 
A készített webalkalmazás ezekre a nehézségekre kínál \textbf{strukturált, digitális megoldást}, amely egységesíti a nyilvántartási és jóváhagyási folyamatokat.

\subsection{A rendszer célja}

A rendszer alapvető célja, hogy \textbf{digitalizálja és automatizálja az informatikai eszközök kezelését}, valamint megkönnyítse a különböző szerepkörű felhasználók munkáját.  
Ennek érdekében a fejlesztés során három fő célkitűzést határoztam meg:

\begin{enumerate}
	\item \textbf{Egységes eszköznyilvántartás létrehozása}  
	Az összes informatikai eszköz (számítógépek, mobiltelefonok, SIM-kártyák stb.) adatai egy központi adatbázisban kerülnek tárolásra. 
	Az eszközök státusza, helye és felelőse valós időben követhető.
	
	\item \textbf{Automatizált eszközrendelési és -jóváhagyási folyamat kialakítása}  
	A dolgozók online rendelhetnek új eszközöket, amelyeket a kiválasztott vezető hagyhat jóvá.
	Ez a digitális folyamat helyettesíti a korábbi e-mailes vagy szóbeli engedélyezéseket.
	
	\item \textbf{Karbantartható és biztonságos rendszer fejlesztése}  
	A jogosultságkezelés több szintű: a rendszer megkülönbözteti a \textit{felhasználó}, a \textit{csoportvezető} és az \textit{adminisztrátor} szerepköröket.  
	Minden felhasználó csak a saját jogosultsági körébe tartozó funkciókat érheti el.
\end{enumerate}

\subsection{Funkcionális követelmények}

A funkcionális követelmények az alkalmazás azon szolgáltatásait írják le, amelyek a felhasználók számára közvetlenül elérhetők.  
A fejlesztés során az alábbi fő funkciók kerültek megvalósításra:

\paragraph{Felhasználói funkciók}
\begin{itemize}
	\item Bejelentkezés az alkalmazásba (JWT token alapú hitelesítés).
	\item Saját adatok és eszközök megtekintése.
	\item Új eszközrendelés létrehozása (pl. számítógép, mobiltelefon, SIM-kártya).
	\item Saját rendelések állapotának nyomon követése.
	\item Az eszköz visszaadásának vagy cseréjének kezdeményezése.
\end{itemize}

\paragraph{Csoportvezetői funkciók}
\begin{itemize}
	\item A beosztottak által létrehozott eszközrendelések megtekintése.
	\item A rendelések \textbf{jóváhagyása vagy elutasítása}.
\end{itemize}

\paragraph{Adminisztrátori funkciók}
\begin{itemize}
	\item Az összes eszköz, rendelés és felhasználó teljes körű kezelése.
	\item Új eszközök felvétele az adatbázisba, kategóriák és státuszok karbantartása.
	\item Rendelések kezelése (eszközök kiadása, visszavétele, státuszok módosítása).
	\item Felhasználók kezelése.
\end{itemize}

\subsection{Nem-funkcionális követelmények}

A rendszer nem-funkcionális követelményei közé tartoznak azok a jellemzők, amelyek a működés minőségét biztosítják:

\begin{itemize}
	\item \textbf{Biztonság:} A kommunikáció JWT token alapú hitelesítéssel történik, ezzel biztosítva az adatokhoz való hozzáférés korlátozását.
	\item \textbf{Teljesítmény:} A .NET 8 és a React technológiák révén az alkalmazás gyors válaszidőt biztosít még nagyobb adatbázis esetén is.
	\item \textbf{Reszponzivitás:} A frontend mobilbarát kialakítású, így bármilyen eszközről kényelmesen használható.
	\item \textbf{Karbantarthatóság:} A komponensalapú React frontend és a rétegzett .NET architektúra biztosítja a könnyű bővíthetőséget.
	\item \textbf{Megbízhatóság:} A fejlesztés során az API végpontokat tesztelésekkel és hibakezeléssel is megerősítettem.
\end{itemize}

\subsection{Összegzés}

A fejlesztett alkalmazás célja, hogy egy \textbf{modern, biztonságos és felhasználóbarát rendszer} segítségével támogassa a vállalat informatikai eszközeinek kezelését.  
Az automatizált folyamatok és az egyértelmű szerepkörök révén a rendszer \textbf{csökkenti az adminisztrációs terheket}, miközben \textbf{növeli az átláthatóságot és a hatékonyságot} a szervezeten belül.
