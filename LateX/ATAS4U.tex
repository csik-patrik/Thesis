\section{A téma aktualitása és célja}

Kis és középvállalatoknál végzett szakmai munkáim során gyakran azt tapasztaltam, hogy a cégnél használt IT eszközök (például számítógépek, laptopok, mobiltelefonok stb.) nem megfelelő nyilvántartása megnehezíti a mindennapi feladatok elvégzését az IT osztályon belül és esetlegesen anyagi károk is jelenthet a munkaadó számára.

A digitalizáció a mai munkaerőpiacon alapkövetelménynek tekintető, ennek következtében az intézményekben használt korábban említett eszközök darabszáma is drasztikusan megnőtt. Az igények folyamatos változása és a használatban lévő eszközök darabszámának növekedése miatt ezeknek az IT eszközöknek a nyilvántartása létfontosságú a vállalati életben.

Tapasztalataim alapján jellemző a különböző táblázatkezelő alkalmazások használata (például Microsoft Excel), melyeknek hátránya, hogy az adott felhasználó minden adatot szabadon tud szerkeszteni, a meglévő adatstruktúrát módosíthatja. Egy közepes méretű cég esetén ez akár több száz különböző eszközt is jelenthet, amiknek a mozgását, státuszát és felelősét nyilvántartani egy táblázatkezelőben nem fenntartható hosszú távon. Az ilyen megoldások alkalmazásakor gyakori az adatvesztés, idővel a struktúra teljes összeomlása és az ,,Egyéb'' mezők rendszeres megjelenése az ,,adatbázisban''.

Szakdolgozatom célja, egy olyan webalkalmazás fejlesztése ami a korábban említett problémákra biztosít egy lehetséges megoldást.
A program egy olyan környezetet szolgáltat a felhasználóknak és az IT munkatársaknak amin keresztül a felhasználók egyszerűen tudnak új eszközöket igényelni, megtekinthetik a jelenlegi eszközeiket és állapotukat, a vezetők jóváhagyási folyamaton keresztül tudják felülvizsgálni az alkalmazottak igényeit, az adminisztrátorok pedig teljes körű rálátást kapnak a telephelyen lévő összes eszközre.

Az alkalmazásom által az eszközök teljes életciklusa digitálisan nyomon követhetővé válik, ezáltal csökken az adminisztrációs teher és minimalizálható az emberi hibákból eredő adatvesztés.

A tervezés során az alábbi célokat tűztem ki:

\begin{itemize}
	\item Eszközök (számítógépek, telefonok, SIM kártyát) kezelése
	\item Igénylési folyamat megvalósítása
	\item Jóváhagyási folyamat megvalósítása
	\item Igénylés feldolgozásának megvalósítása
	\item Átadási / Átvételi dokumentum digitalizációja
	\item Használatban lévő eszközök életciklusának követhetősége
	\item Felhasználó és jogosultság kezelés
\end{itemize}

Célom, hogy az alkalmazásom bevezetésével növelem az átláthatóságot és hosszú távon csökkenthetem az üzemeltetésre, adminisztrációra fordítót kapacitás mennyiségét.
A fejlesztés során személyes célom, hogy elmélyítsem a tudásom a piacképes technológiák használatában és az alkalmazásfejlesztés vállalati szintű gyakorlatában.

\section{A használt technológiák bemutatása}
A munkám során szerettem volna olyan technológia eszközöket alkalmazni, melyek illeszkednek a jelenlegi iparági trendekhez és megfelelnek a vállalati fejlesztési eszközök elvárásainak.

A backend fejlesztéshez a .NET 8 Web API keretrendszert alkalmaztam, amely erősen típusos, jól strukturált és magas teljesítményű környezetet biztosít. Ezzel szemben a Node.js alapú Express.js könnyebb és gyorsabban tanulható megoldás, azonban nagyobb alkalmazások esetén a strukturáltság hiánya miatt nehezebb lehet a karbantartása. A Java alapú Spring Boot hasonlóan robusztus és vállalati környezetben elterjedt alternatíva, azonban konfigurációja és komplexitása általában nagyobb, mint a .NET megoldásé.

A frontend oldalon a React könyvtár került alkalmazásra, amely komponensalapú megközelítést és rugalmas felépítést biztosít. Az Angular ezzel szemben egy teljes körű keretrendszer, amely sok beépített funkcióval rendelkezik, ugyanakkor komplexebb és kötöttebb struktúrát követel meg. A Vue.js egy könnyebben tanulható alternatíva, amely gyors fejlesztést tesz lehetővé, azonban nagyobb projektek esetén kevésbé biztosít olyan szintű strukturáltságot, mint a React vagy az Angular.

Az adatok tárolására Microsoft SQL Szervert alkalmaztam, amely jól kezeli az adatok közötti kapcsolatokat és biztosítja az adatintegritást. Ezzel szemben a NoSQL adatbázisok, például a MongoDB, rugalmasabb sémakezelést kínálnak, és nagy mennyiségű, strukturálatlan adat esetén előnyösebbek lehetnek. A jelen rendszer esetében azonban a relációs megközelítés bizonyult megfelelőbbnek a jól definiált entitáskapcsolatok miatt.

Összességében elmondható, hogy a választott technológiák stabil, jól skálázható és karbantartható rendszert eredményeznek, miközben megfelelő alternatívát jelentenek a hasonló célokra használt más népszerű technológiákkal szemben.

\subsection{Microsoft .NET 8 Web API}

A szerveroldali logika megvalósításához a .NET 8 Web API keretrendszert alkalmaztam.

A .NET egy ingyenes, platformfüggetlen, nyílt forráskódú fejlesztői platform, amely különféle alkalmazások építésére nyújt lehetőséget. 
A nagy teljesítményű futtatókörnyezetre támaszkodik, amelyet számos nagy léptékű alkalmazás használ éles környezetben.
A .NET platform a termelékenység, a teljesítmény, a biztonság és a megbízhatóság biztosításához készült. \cite{DotnetBevezetés}

A rendszer az adatbázissal való kommunikáció során az Entity Framework Core objektum-relációs leképező (ORM) keretrendszert használtam. Ez lehetővé teszi, hogy az adatbázis műveletek objektumorientált módon, C\# nyelven kerüljenek megvalósításra, így elkerülve a kézzel írt SQL lekérdezéseket amely csökkenti a lehetséges hibák számát.

Az Entity Framework (EF) Core a népszerű Entity Framework adatelérési technológia egyszerűsített, bővíthető, nyílt forráskódú és platformfüggetlen verziója.

Az EF Core objektum--relációs leképezőként (O/RM) szolgálhat, amely:

\begin{enumerate}
	\item Lehetővé teszi, hogy a .NET-fejlesztők .NET-objektumokkal dolgozzanak egy adatbázissal.
	\item Kiküszöböli az általában írandó adatelérési kódok többségének szükségességét.
\end{enumerate}

Az EF Core használatával az adathozzáférés egy modell használatával történik. A modell entitásosztályokból és környezeti objektumból áll, amelyek az adatbázissal való munkamenetet jelölik. A környezeti objektum lehetővé teszi az adatok lekérdezését és mentését.\cite{EntityFrameworkCore}

\subsection{Microsoft SQL Server}
Az alkalmazás adatainak tárolására a Microsoft SQL Server relációs adatbázis-kezelő rendszert alkalmaztam. A választás során fontos szempont volt a megbízhatóság, a teljesítmény, valamint a .NET környezettel való szoros integráció.

A Microsoft SQL Server egy széles körben elterjedt, vállalati szintű adatbázis-kezelő rendszer, amely támogatja a strukturált adatok hatékony tárolását és kezelését. A relációs modell alkalmazása lehetővé teszi az adatok közötti kapcsolatok egyértelmű definiálását, amely különösen fontos az eszközök, felhasználók és rendelések közötti összefüggések kezelésében.

Az adatbázis tervezése során normalizált adattáblákat hoztam létre, amelyek csökkentik az adatredundanciát és biztosítják az adatok konzisztenciáját. Az egyes entitások közötti kapcsolatokat idegen kulcsok segítségével valósítottam meg, amely biztosítja az adatintegritást.

Alternatív megoldásként szóba jöhettek volna más relációs adatbázis-kezelők, például a MySQL vagy a PostgreSQL. Ezek szintén megbízható és széles körben használt rendszerek, azonban a Microsoft SQL Server előnye a .NET környezettel való natív integráció, valamint a fejlesztői eszközök (pl. SQL Server Management Studio) által nyújtott kényelmes használat.

A Microsoft SQL Server a Microsoft által fejlesztett, zárt forráskódú relációs adatbázis-kezelő rendszer, amely a Structured Query Language (SQL) nyelvet használja. Adatbázis-szerverként működve elsődleges feladata az adatok tárolása és lekérdezése más szoftveralkalmazások kérésére. Ezek az alkalmazások futhatnak ugyanazon a számítógépen, vagy hálózaton keresztül -- beleértve az internetet is -- más számítógépeken.\cite{SQLServer}

Összességében a Microsoft SQL Server használata stabil, jól skálázható és megbízható adatkezelést biztosít az alkalmazás számára, miközben hatékonyan támogatja a backend rendszer működését.

\subsection{React}

A frontend fejlesztése során a React JavaScript könyvtárat alkalmaztam, amely napjaink egyik legnépszerűbb megoldása dinamikus felhasználói felületek kialakítására. A React komponensalapú megközelítést alkalmaz, amely lehetővé teszi az alkalmazás felületének kisebb, újrafelhasználható egységekre bontását. Ez a struktúra megkönnyíti a fejlesztést, a karbantartást, valamint a kód újrafelhasználását.

A React egyik legfontosabb előnye a virtuális DOM (Document Object Model) használata, amely optimalizálja a felhasználói felület frissítését. Ennek köszönhetően csak azok az elemek kerülnek újrarenderelésre, amelyek ténylegesen megváltoztak, így javul az alkalmazás teljesítménye.

A fejlesztés során React hook-okat (például useState és useEffect) használtam az állapotkezelés és az életciklus események kezelésére. Ez lehetővé teszi a funkcionális komponensek használatát, amelyek egyszerűbb és átláthatóbb kódot eredményeznek.

Alternatív megoldásként szóba jöhetett volna az Angular keretrendszer, amely egy teljes körű frontend megoldást kínál. Az Angular számos beépített funkcióval rendelkezik, például dependency injection, routing és formkezelés, így egy komplexebb, strukturáltabb környezetet biztosít. Ugyanakkor a fejlesztés során több előre definiált szabályhoz kell igazodni.

A React ezzel szemben nagyobb rugalmasságot biztosít, mivel nem egy teljes keretrendszer, hanem egy könyvtár, amelyhez a fejlesztő választhatja ki a szükséges kiegészítő eszközöket (például routing vagy állapotkezelés). Ez különösen előnyös kisebb vagy közepes méretű projektek esetén, ahol fontos a gyors fejlesztés és az egyszerűség.

Az Angular erőssége inkább nagyvállalati alkalmazások esetén jelentkezik, ahol a szigorúbb struktúra és a beépített megoldások előnyt jelentenek. Ezzel szemben a React könnyebb tanulhatósága és rugalmassága miatt szélesebb körben elterjedt, különösen modern webalkalmazások fejlesztése során.

Összességében a React választása megfelelő döntésnek bizonyult, mivel jól illeszkedik a projekt követelményeihez, és lehetővé teszi egy dinamikus, jól karbantartható felhasználói felület kialakítását.


\subsection{Fejlesztői környezet és eszközök}
A fejlesztés során több, egymással jól együttműködő fejlesztői eszközt alkalmaztam, amelyek hozzájárultak a hatékony és strukturált munkavégzéshez. A backend fejlesztése Microsoft Visual Studio 2022 környezetben történt, míg a frontend fejlesztéshez Visual Studio Code szerkesztőt használtam.

A Microsoft Visual Studio 2022 egy teljes körű integrált fejlesztői környezet (IDE), amely kifejezetten jól támogatja a .NET alapú alkalmazások fejlesztését. Előnyei közé tartozik a fejlett hibakeresési lehetőség (debugging), az automatikus kódkiegészítés (IntelliSense), valamint a beépített eszközök, amelyek megkönnyítik a projekt kezelését és a kód navigációját. A Visual Studio különösen előnyös nagyobb backend projektek esetén, ahol fontos a stabilitás és a komplex funkcionalitás támogatása.

A frontend fejlesztéshez a Visual Studio Code szerkesztőt alkalmaztam, amely egy könnyű, gyors és bővíthető eszköz. A Visual Studio Code támogatja a különböző programozási nyelveket és keretrendszereket, és számos kiegészítő segítségével testre szabható. A React fejlesztés során különösen hasznosak voltak az olyan bővítmények, amelyek támogatják a JavaScript és TypeScript fejlesztést, valamint az automatikus kódformázást.

Alternatív fejlesztői környezetként szóba jöhetett volna például a JetBrains által fejlesztett IntelliJ IDEA vagy WebStorm, amelyek szintén széles körű funkcionalitást biztosítanak. Ezek az eszközök különösen kiemelkedőek frontend fejlesztésben, azonban használatuk licenchez kötött. Ezzel szemben a Visual Studio Code ingyenesen elérhető, és megfelelő funkcionalitást nyújt a legtöbb fejlesztési feladathoz.

A verziókezeléshez Git rendszert alkalmaztam, amely lehetővé teszi a kód változásainak nyomon követését és a verziók kezelését. A forráskód tárolása GitHub platformon történt, amely biztosítja a kód biztonságos tárolását és az együttműködés lehetőségét.

A fejlesztés során az API végpontok tesztelésére Postman alkalmazást használtam. Ez az eszköz lehetővé teszi HTTP kérések egyszerű küldését és a válaszok részletes vizsgálatát, ami különösen hasznos a backend fejlesztés során.

Összességében a választott fejlesztői környezet és eszközök jól támogatják a modern webalkalmazások fejlesztését, és biztosítják a hatékony, átlátható és jól szervezett munkafolyamatot.

\chapter{A rendszer célja és funkcionális követelményei}

Az alkalmazásom célja, hogy a vállalati környezetben használt informatikai eszközöknek (jelenleg számítógépek, laptopok, telefonok, SIM kártyák) biztosítson egy átlátható és könnyen karbantartható felületet.

A megvalósítás során fontos szempont volt, hogy az alkalmazás egyszerűen használható, a jövőbeli igényeknek megfelelően bővíthető legyen.

Elvárt követelmény, hogy az alkalmazás bevezetése után a felhasználók adminisztrációs feladatait csökkenti, kapacitást szabadít fel.

\section{A rendszer célja}

A rendszer alapvető célja, hogy \textbf{digitalizálja és automatizálja az informatikai eszközök nyilvántartását}, valamint megkönnyítse a felhasználók munkáját.
Ennek érdekében a fejlesztés során az alábbi célokat határoztam meg:

\begin{enumerate}
	\item Egységes eszköznyilvántartás létrehozása
	
	Az összes informatikai eszköz adatai egy központi adatbázisban kerülnek tárolásra.
	Az eszközről tárolt információ, jelenlegi állapota és felelőse valós időben követhető.
	
	\item Eszközrendelési és jóváhagyási folyamat kialakítása 
	
	A dolgozók az alkalmazáson keresztül adhatják le új eszköz igényüket, amelyet a kiválasztott vezető bírál el.
	A rendelések feldolgozását az adminisztrátorok végzik.
	A rendelés feldolgozásához szükséges információkat és eszközöket egy felületen érnek el.
	Ez a digitális folyamat helyettesíti a korábbi e-mailes vagy szóbeli engedélyezéseket.
	
	\item Karbantartható és biztonságos rendszer fejlesztése
	
	A jogosultságkezelés több szintű: a rendszer megkülönbözteti a \textit{felhasználó}, a \textit{vezető} és az \textit{adminisztrátor} szerepköröket.
	
	Minden felhasználó csak a saját jogosultsági körébe tartozó funkciókat érheti el.
\end{enumerate}

\section{Funkcionális követelmények}

A rendszer funkcionális követelményei meghatározzák azokat a műveleteket és szolgáltatásokat, amelyeket az alkalmazásnak biztosítania kell a felhasználók számára. A követelmények meghatározása során figyelembe vettem a rendszer célját, valamint a különböző felhasználói szerepkörök igényeit.

A rendszer többféle felhasználói szerepkört különböztet meg, például normál felhasználót, vezetőt és adminisztrátort. Az egyes szerepkörök eltérő jogosultságokkal rendelkeznek, amelyek meghatározzák, hogy milyen műveleteket hajthatnak végre az alkalmazásban.

A legfontosabb funkcionális követelmények a következők:

\subsection{Bejelentkezés és hitelesítés}

A rendszer használata kizárólag hitelesített felhasználók számára engedélyezett. Ennek megfelelően az alkalmazás minden funkciója csak bejelentkezést követően érhető el.

A felhasználók a bejelentkezési felületen e-mail címük és jelszavuk megadásával azonosítják magukat. A megadott adatok ellenőrzését a szerver végzi.

Sikeres hitelesítés esetén a rendszer hozzáférést biztosít a felhasználó számára a jogosultsági szintjének megfelelő funkciókhoz. Sikertelen bejelentkezés esetén a rendszer hibaüzenetet jelenít meg, és nem engedélyezi a további műveletek végrehajtását.

A bejelentkezési folyamat biztosítja, hogy az alkalmazás funkciói kizárólag jogosult felhasználók számára legyenek elérhetők.

\subsection{Felhasználó}

\begin{itemize}
	\item Eszközigénylés létrehozása

		Mobil eszköz igénylése esetén választható mező, amely az adminisztrátorok által definiált értékeket tartalmazza. Az egyes ügyféltípusok eltérő előfizetési csomagokat jelentenek.

	\end{itemize}
	
	Az igénylés rögzítését követően a felhasználó visszajelzést kap a felületen, továbbá értesítés kerül kiküldésre az igénylő és a kijelölt jóváhagyó részére.

	A felhasználók a számukra elérhető felületen ellenőrizhetik a korábban leadott igénylések adatait, valamint azok aktuális státuszát.

\section{JSON Web Token működése}

A JSON Web Token (JWT) egy nyílt szabvány (RFC 7519), amely kompakt, önmagát tartalmazó módot biztosít a felek közötti biztonságos információcserére JSON objektumként. Gyakran használják modern, állapot nélküli (stateless) webes alkalmazásokban hitelesítésre (authentication) és engedélyezésre (authorization), mivel a token magában hordozza a felhasználói adatokat, aláírása révén pedig hamisíthatatlan. \cite{JWT} 

A token felépítése:
\begin{itemize}
	\item Header
	
	A header rész általában két elemet tartalmaz, a token típusát és az alkalmazott aláírási algoritmust például HMAC SHA256 vagy RSA.
		
	\item Payload
	
	A token második része a payload, amely az úgynevezett állításokat (claims) tartalmazza. Az állítások egy adott entitásra -- jellemzően a felhasználóra -- vonatkozó információkat, valamint további adatokat foglalnak magukba. Az állítások három fő típusa különböztethető meg: regisztrált (registered), nyilvános (public) és privát (private) állítások.
	
	\item Signature
	
	Az aláírás létrehozásához a titkosított fejlécet, a titkosított payloadot, egy titkos kulcsot, valamint a fejlécben megadott algoritmust kell felhasználni, majd ezek alapján történik az aláírás generálása.
	
\end{itemize}

\section{Integráció}
A frontend és a backend közötti integráció REST API-n keresztül valósul meg. A kliensoldali alkalmazás HTTP kéréseket küld a backend végpontjai felé, amelyek feldolgozzák a kéréseket és visszaküldik a szükséges adatokat.

A rendszer biztosítja a hibakezelést is, amennyiben egy API hívás sikertelen, a frontend megfelelő visszajelzést ad a felhasználónak.

Az integráció során külön figyelmet fordítottam arra, hogy a frontend és backend közötti adatcsere egységes és jól strukturált legyen.

\section{Biztonság és naplózás}
A rendszer biztonságos működése érdekében több védelmi mechanizmust alkalmaztam.

A felhasználók hitelesítése JWT token alapú megoldással történik, amely biztosítja, hogy csak jogosult felhasználók férhessenek hozzá az API végpontokhoz.

A jelszavak nem kerülnek tárolásra nyílt formában, hanem titkosított (hash-elt) módon kerülnek az adatbázisba. Ez jelentősen növeli az adatok biztonságát.

A rendszer tartalmaz alapvető hibakezelést is, amely biztosítja, hogy a váratlan hibák ne okozzanak rendszerleállást.

A naplózás (logging) lehetővé teszi a rendszer működésének nyomon követését, valamint a hibák későbbi elemzését. A naplózott események közé tartozhatnak például a bejelentkezések, hibák és fontosabb műveletek.

\section{Tesztelési módszertan}
A fejlesztett rendszer megfelelő működésének biztosítása érdekében különböző tesztelési módszereket alkalmaztam. A tesztelés célja az volt, hogy ellenőrizzem az alkalmazás funkcionalitását, megbízhatóságát és stabilitását.

A tesztelés során elsősorban manuális tesztelést végeztem, amely során a rendszer különböző funkcióit valós felhasználói szemszögből próbáltam ki. Emellett az API végpontok működését külön is ellenőriztem Postman segítségével.

Az alábbi tesztelési típusokat alkalmaztam:

\begin{itemize}
	\item Funkcionális tesztelés:
	Annak ellenőrzése, hogy az egyes funkciók (pl. bejelentkezés, eszköz rendelés, jóváhagyás) megfelelően működnek-e.
	\item Integrációs tesztelés:
	A frontend és a backend közötti kommunikáció vizsgálata, különös tekintettel az API hívások helyes működésére.
	\item Hibakezelés tesztelése:
	Annak ellenőrzése, hogy a rendszer hogyan reagál hibás bemenetekre vagy sikertelen műveletekre.
	\item Jogosultság tesztelése:
	A különböző szerepkörök (felhasználó, vezető, adminisztrátor) jogosultságainak ellenőrzése.
\end{itemize}

A tesztelés során törekedtem arra, hogy minden fontos funkciót több különböző esettel is lefedjek.

\chapter{További fejlesztési lehetőségek}

A fejlesztett alkalmazás jelenlegi formájában jól használható megoldást nyújt az informatikai eszközök nyilvántartására és kezelésére, azonban számos olyan fejlesztési lehetőség megvalósítható, amelyek tovább növelhetik a rendszer hatékonyságát és felhasználói élményt.

Az egyik legfontosabb jövőbeli fejlesztési irány az e-mail alapú értesítési rendszer bevezetése. Ennek segítségével a felhasználók automatikus értesítést kaphatnának például új eszközrendelés létrehozásakor, annak jóváhagyásáról vagy elutasításáról, illetve eszköz visszavétel esetén. Ez jelentősen javítaná a kommunikáció hatékonyságát, és csökkentené annak esélyét, hogy egy-egy fontos esemény elkerülje a felhasználók figyelmét.

A rendszer funkcionalitása bővíthető lenne egy statisztikai és riportáló modul bevezetésével is. Ennek segítségével a vezetők különböző kimutatásokat készíthetnének, például az eszközök kihasználtságáról, a rendelések számáról vagy az egyes eszköztípusok eloszlásáról. Ezek az adatok támogatnák a stratégiai döntéshozatalt.

Egy másik lehetséges fejlesztés a mobilalkalmazás létrehozása, amely lehetővé tenné a rendszer használatát okostelefonról is. Ez különösen hasznos lehet olyan környezetben, ahol a felhasználók gyakran dolgoznak irodán kívül.

A biztonság tovább növelhető lenne kétfaktoros hitelesítés (2FA) bevezetésével, amely további védelmi réteget biztosít a felhasználói fiókok számára.

Végül a rendszer bővíthető lenne külső rendszerekkel való integrációval, például vállalati címtárakkal (pl. Active Directory), amely lehetővé tenné az automatikus felhasználókezelést és az egységes hitelesítést.

Összességében elmondható, hogy a fejlesztett alkalmazás jó alapot biztosít további funkciók bevezetéséhez, és megfelelő továbbfejlesztéssel egy komplex, vállalati szintű eszközkezelő rendszerré alakítható.

\chapter{Összefoglalás és köszönetnyilvánítás}

A szakdolgozat célja egy olyan webalapú alkalmazás megtervezése és megvalósítása volt, amely támogatja az informatikai eszközök nyilvántartását, valamint az eszközrendelési és jóváhagyási folyamatok kezelését. A dolgozat során egy olyan rendszer került kialakításra, amely képes a vállalati környezetben gyakran előforduló adminisztrációs problémák hatékony kezelésére.

A fejlesztés során modern technológiákat alkalmaztam mind a backend, mind a frontend oldalon. A backend megvalósítása .NET 8 Web API keretrendszerrel történt, míg a felhasználói felület React alapokon került kialakításra. Az adatkezelést relációs adatbázis biztosítja, amely lehetővé teszi az adatok közötti kapcsolatok strukturált és megbízható kezelését.

A dolgozat részletesen bemutatta a rendszer tervezését, beleértve az architektúra kialakítását és az adatbázis megtervezését, valamint ismertette a megvalósítás során alkalmazott technológiai megoldásokat. A tesztelési fejezet igazolta, hogy az alkalmazás stabilan működik és megfelel a meghatározott funkcionális követelményeknek.

A fejlesztés során szerzett tapasztalatok hozzájárultak a modern webalkalmazások fejlesztéséhez szükséges ismereteim elmélyítéséhez, különös tekintettel a kliens–szerver architektúrára, a REST API-k kialakítására, valamint a jogosultságkezelés megvalósítására.

Összességében elmondható, hogy a kitűzött célokat sikerült megvalósítani, és egy működőképes, jól bővíthető rendszert hoztam létre. A dolgozatban bemutatott megoldás megfelelő alapot biztosít további fejlesztésekhez, és a jövőben akár komplexebb vállalati környezetben is alkalmazható.
